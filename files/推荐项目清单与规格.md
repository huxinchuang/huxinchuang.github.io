# 推荐项目清单与规格

## 项目选择原则
1. **循序渐进**：从简单到复杂，覆盖数字IC设计主要环节
2. **实用价值**：项目内容与实际工作相关，可用于作品集展示
3. **技术全面**：涵盖设计、验证、脚本、低功耗等技能
4. **协议标准**：使用工业界标准协议（如AXI、APB）
5. **开源友好**：可在GitHub公开，不涉及公司机密

## 项目一：ALU（算术逻辑单元）
**难度**：★☆☆☆☆  
**时间**：1周  
**技术栈**：SystemVerilog设计、基础testbench

### 规格要求
- **位宽**：8位或16位
- **操作**：至少支持加、减、与、或、异或、移位
- **接口**：
  - `input logic [7:0] a, b`
  - `input logic [2:0] opcode`
  - `output logic [7:0] result`
  - `output logic zero, carry, overflow`
- **功能**：
  - 根据opcode执行相应运算
  - 生成标志位（零标志、进位标志、溢出标志）

### 实现要点
1. 使用`always_comb`实现组合逻辑
2. 使用`case`语句根据opcode选择操作
3. 正确计算标志位
4. 编写testbench验证所有操作

### 验证要求
- 编写简单testbench
- 验证所有opcode功能
- 检查边界情况（如溢出）
- 生成波形图

### 扩展建议（可选）
1. 添加流水线寄存器
2. 支持32位操作
3. 添加乘法操作

## 项目二：同步FIFO
**难度**：★★☆☆☆  
**时间**：2周  
**技术栈**：SystemVerilog设计、UVM验证

### 规格要求
- **深度**：8或16
- **宽度**：16位或32位
- **接口**：
  - 时钟与复位：`clk, rst_n`
  - 写端口：`wr_en, wr_data`
  - 读端口：`rd_en, rd_data`
  - 状态信号：`full, empty, almost_full, almost_empty`
- **功能**：
  - 同步时钟域
  - 满时拒绝写入
  - 空时拒绝读取
  - 可配置almost_full/almost_empty阈值

### 实现要点
1. 使用双端口RAM或寄存器数组存储数据
2. 读写指针使用格雷码避免亚稳态
3. 状态信号生成逻辑
4. 参数化设计（深度、宽度可配置）

### 验证要求（UVM环境）
1. 搭建完整UVM环境
2. 编写sequence产生随机读写操作
3. 检查数据一致性（写什么读什么）
4. 验证full/empty边界条件
5. 收集功能覆盖率：
   - 读写同时发生
   - 满状态写入尝试
   - 空状态读取尝试
   - 指针回绕

### 扩展建议（可选）
1. 添加异步FIFO（双时钟域）
2. 支持burst传输
3. 添加性能计数器（吞吐量、延迟）

## 项目三：UART控制器
**难度**：★★★☆☆  
**时间**：2-3周  
**技术栈**：SystemVerilog设计、UVM验证、协议理解

### 规格要求
- **标准**：RS-232兼容，8N1格式（8数据位，无校验，1停止位）
- **波特率**：可配置（9600, 115200等）
- **接口**：
  - 系统接口：APB或AXI-lite（用于配置）
  - 串行接口：`tx, rx`
  - 中断信号：`tx_done, rx_ready`
- **寄存器**（通过APB/AXI访问）：
  - 控制寄存器（使能、波特率选择）
  - 状态寄存器（发送忙、接收满）
  - 发送数据寄存器
  - 接收数据寄存器

### 实现要点
1. 波特率生成器（分频器）
2. 发送状态机：空闲→起始位→数据位→停止位
3. 接收状态机：检测起始位→采样数据位→检测停止位
4. APB/AXI接口逻辑
5. FIFO缓冲（可选，提高性能）

### 验证要求
1. UVM验证环境包含：
   - 寄存器模型（RAL）
   - 虚拟sequence协调发送和接收
   - 协议检查器（检查时序）
2. 验证场景：
   - 不同波特率下的数据传输
   - 连续发送接收
   - 错误注入（帧错误）
3. 覆盖率：
   - 寄存器访问组合
   - 波特率配置
   - 数据模式（全0、全1、随机）

### 扩展建议（可选）
1. 支持硬件流控制（RTS/CTS）
2. 添加DMA传输支持
3. 多通道UART控制器

## 项目四：低功耗睡眠控制器
**难度**：★★★☆☆  
**时间**：2周  
**技术栈**：SystemVerilog、UPF、低功耗设计

### 规格要求
- **电源域**：两个电源域（常开域、可关断域）
- **控制逻辑**：在常开域中
- **数据通路**：在可关断域中
- **接口**：
  - 唤醒信号：`wakeup`
  - 电源控制：`power_on`（控制可关断域电源）
  - 隔离控制：`isolate`（隔离输出信号）
  - 状态输出：`sleep_state`

### 功能描述
1. **正常工作模式**：两个电源域都开启
2. **睡眠进入**：收到sleep信号后，保存关键寄存器，启动隔离，然后关闭可关断域电源
3. **睡眠唤醒**：收到wakeup信号后，开启可关断域电源，解除隔离，恢复寄存器
4. **状态机**：IDLE→SLEEP_ENTRY→SLEEP→WAKE_ENTRY→IDLE

### 实现要点
1. 状态机设计（使用低功耗编码风格）
2. 隔离单元插入（使用`enable`信号控制）
3. 电源开关控制（模拟）
4. 寄存器保存与恢复（使用常开域中的影子寄存器）
5. UPF文件编写：
   ```
   create_power_domain PD_ALWAYS_ON -elements {ctrl_logic}
   create_power_domain PD_SHUTDOWN -elements {data_path}
   create_supply_port VDD -domain PD_SHUTDOWN
   create_power_switch SWITCH -domain PD_SHUTDOWN \
     -input_supply_port {in VDD} \
     -output_supply_port {out VDD_SW} \
     -control_port {ctrl power_on} \
     -on_state {on_state in {ctrl == 1}}
   ```

### 验证要求
1. 功能验证：状态转换正确性
2. 低功耗验证（使用支持UPF的仿真工具）：
   - 检查电源开关控制信号
   - 验证隔离信号时序
   - 检查无电流泄漏路径
3. 综合与功耗分析（使用Design Compiler + Power Compiler）

### 扩展建议（可选）
1. 多电压域设计
2. 动态电压频率调整（DVFS）
3. 电源状态机更复杂（多种睡眠模式）

## 项目五：AXI-lite to APB Bridge
**难度**：★★★★☆  
**时间**：3周  
**技术栈**：SystemVerilog、UVM、协议转换、系统集成

### 规格要求
- **AXI-lite接口**：符合AMBA AXI-lite协议
  - 地址通道：AW, W, AR
  - 响应通道：B, R
- **APB接口**：符合AMBA APB协议
  - 标准APB 3.0接口：PCLK, PRESETn, PADDR, PSEL, PENABLE, PWRITE, PWDATA, PRDATA
- **功能**：
  - 将AXI-lite读写转换为APB读写
  - 支持32位地址和数据
  - 错误处理（AXI错误响应）
  - 可配置延迟（模拟APB访问延迟）

### 架构设计
```
AXI-lite Master → Bridge → APB Slave
                  ↓
            Control FSM
                  ↓
           Address Decoder
                  ↓
           Data Width Converter (可选)
```

### 实现要点
1. 状态机：处理AXI请求，生成APB时序
2. 地址映射：支持多个APB slave选择（通过PSEL）
3. 数据对齐：处理不同位宽转换（如果AXI 64位，APB 32位）
4. 错误处理：APB错误时返回AXI SLVERR
5. 性能优化：支持背靠背传输（pipelining）

### 验证要求（完整UVM环境）
1. **验证组件**：
   - AXI-lite VIP（或自制driver/monitor）
   - APB VIP（或自制driver/monitor）
   - 记分板：检查AXI与APB事务一致性
   - 覆盖率收集器
2. **测试场景**：
   - 基本读写操作
   - 背靠背传输
   - 错误注入（APB错误响应）
   - 地址边界测试
   - 随机延迟测试
3. **覆盖率**：
   - AXI与APB事务类型组合
   - 地址对齐情况
   - 延迟配置组合
   - 错误响应路径

### 扩展建议（可选）
1. 添加AXI-lite从接口和APB主接口（反向桥接）
2. 支持AXI-lite批量传输（burst）
3. 添加性能监控计数器
4. 支持安全扩展（TrustZone）

## 项目六：小型RISC-V SoC
**难度**：★★★★★  
**时间**：4-6周  
**技术栈**：SystemVerilog、UVM、CPU架构、系统集成

### 系统架构
```
+----------------+    AXI-lite    +----------------+
| RISC-V CPU     |<-------------->| AXI-APB Bridge |
| (RV32IMC)      |                +----------------+
+----------------+                       |
     |                             APB Bus
     |                                 |
+----------------+    +----------------+----------------+
| Instruction    |    | UART   | Timer  | GPIO  | ...  |
| Memory (IMEM)  |    +----------------+----------------+
+----------------+
| Data Memory    |
| (DMEM)         |
+----------------+
```

### 组件规格
1. **RISC-V CPU核心**：RV32IMC（整数、乘除、压缩指令集）
   - 3-5级流水线
   - 简单的分支预测
   - 中断支持（可选）
2. **存储器**：
   - IMEM：32KB指令存储器（ROM或RAM）
   - DMEM：32KB数据存储器（RAM）
   - 存储器映射：统一地址空间
3. **外设**：
   - UART控制器（使用项目三）
   - 定时器（32位向上计数器，可产生中断）
   - GPIO（32位输入输出）
   - 中断控制器（PLIC简化版）

### 实现要点
1. CPU微架构设计
2. 总线互联（AXI-lite或自定义总线）
3. 外设集成与地址映射
4. 系统启动流程（从固定地址取指）
5. 调试接口（可选，如JTAG）

### 验证要求
1. **分层验证**：
   - 模块级：CPU核心、外设独立验证
   - 子系统级：总线互联验证
   - 系统级：整体功能验证
2. **测试方法**：
   - 运行RISC-V测试套件（riscv-tests）
   - 运行简单C程序（通过UART输出）
   - 性能测试（Dhrystone benchmark）
3. **覆盖率**：
   - 指令覆盖率（所有RISC-V指令）
   - 外设访问组合
   - 中断处理路径

### 扩展建议（可选）
1. 添加缓存（I-Cache, D-Cache）
2. 支持多核（双核简单互连）
3. 添加DMA控制器
4. 运行Linux（需要MMU和更多外设）

### 代码质量（40%）
- 代码风格一致（命名规范、缩进）
- 注释清晰（模块说明、接口描述）

- 验证环境可重用性
- 回归测试自动化

- README说明项目背景和使用方法
