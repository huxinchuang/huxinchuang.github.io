# 济南旅游规划网站开发计划

## 项目概述
创建一个有趣、实用的济南旅游规划网站，小团体内部使用。核心功能：地图展示、地址搜索、标记点记录、路线规划。

## 技术栈
- HTML/CSS/JavaScript
- 高德地图API（适合中国地区）
- Tailwind CSS（与现有站点风格一致）
- **Firebase Firestore**（多人实时协作）
- LocalStorage（离线降级方案）
- Font Awesome图标

## 功能模块

### 1. 地图模块
- 高德地图集成
- 济南市中心默认视图
- 缩放、平移控制
- 地图样式定制（旅游主题）

### 2. 搜索模块
- 地址搜索框
- 自动补全建议
- 搜索结果在地图标记
- 搜索历史记录

### 3. 标记点管理
- 点击地图添加标记
- 标记点信息编辑（名称、描述、类别）
- 标记点颜色分类（景点、美食、住宿等）
- 标记点拖拽调整位置
- 标记点删除

### 4. 路线规划
- 标记点连线生成路线
- 路线顺序调整
- 路线时间估算
- 路线导出/分享

### 5. 数据管理
- LocalStorage保存所有数据
- 数据导入/导出（JSON格式）
- 数据重置功能

### 6. 创意功能
- 泉水收集游戏化界面
- 景点打卡系统
- 旅游统计面板
- 分享截图功能

## 部署说明

### 1. 高德地图API配置
1. 访问高德地图开放平台 (https://lbs.amap.com/)
2. 注册账号并创建应用
3. 获取Key
4. 在 `jinan-travel.html` 第 12 行替换 `你的高德Key` 为实际 Key

### 2. Firebase配置
1. 访问 Firebase Console (https://console.firebase.google.com/)
2. 创建项目（免费 Spark 计划即可）
3. 在项目设置中获取 Web App 的配置信息
4. 在 `jinan-travel.html` 中替换 `FIREBASE_CONFIG` 中的占位值：
   - `apiKey`、`authDomain`、`projectId`、`storageBucket`、`messagingSenderId`、`appId`
5. 在 Firestore 中创建集合 `jinan_travel` 和 `jinan_travel_presence`

### 3. Firestore安全规则
建议设置以下安全规则（允许所有已认证用户读写）：
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```
> 注意：小团体内部使用可暂时开放，生产环境需配置更严格的规则。

### 2. 功能说明
- **地图操作**: 缩放、平移、点击添加标记
- **搜索功能**: 地址搜索、结果展示
- **标记点管理**: 添加、编辑、删除、分类
- **数据管理**: 本地存储、导入导出、重置
- **路线规划**: 基础框架（需进一步开发）

### 3. 使用流程
1. 打开 `jinan-travel.html`
2. 搜索地点或点击地图添加标记点
3. 为标记点填写信息并选择类别
4. 使用路线规划功能连接标记点
5. 导出数据与团队成员分享

### 4. 扩展建议
- 添加路线优化算法
- 集成公共交通路线
- 添加时间规划功能
- 实现多人协作编辑
- 添加景点图片上传

### 5. 数据管理
- **Firebase Firestore**: 多人实时同步，所有人可见
- **LocalStorage**: Firebase不可用时自动降级为本地存储
- **导入/导出**: 支持JSON格式导出分享
- **在线用户**: 实时显示当前在线协作人数

### 6. 多人协作功能
- **用户身份**: 进入时输入昵称，显示在标记点上
- **实时同步**: Firestore `onSnapshot` 实时监听数据变化
- **在线状态**: 30秒心跳更新，1分钟超时自动离线
- **防冲突**: 本地与远程数据对比，避免重复刷新
